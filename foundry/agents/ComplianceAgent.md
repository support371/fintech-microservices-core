# ComplianceAgent

## Purpose

The Compliance Agent enforces regulatory requirements across the Nexus platform.
It validates KYC (Know Your Customer) tiers, performs AML (Anti-Money Laundering)
screening, and ensures every transaction meets the compliance policies defined by
Alliance Trust Realty.

## Responsibilities

- **KYC Tier Enforcement** — Verify that users meet the minimum KYC tier (≥ 3)
  before card issuance or fund operations.
- **AML Screening** — Flag transactions that exceed configurable thresholds or
  match suspicious patterns.
- **Audit Trail** — Emit structured compliance events for every decision so the
  ledger agent can record them.
- **Regulatory Reporting** — Generate periodic compliance summaries for internal
  review.

## Inputs

| Field           | Type   | Source                        |
|-----------------|--------|-------------------------------|
| `user_id`       | string | Request payload               |
| `kyc_tier`      | int    | PostgreSQL `users` table      |
| `fiat_amount`   | number | Transaction request           |
| `fiat_currency` | string | Transaction request           |
| `trace_id`      | string | Generated by card service     |

## Outputs

| Field              | Type    | Description                          |
|--------------------|---------|--------------------------------------|
| `approved`         | boolean | Whether the operation is permitted   |
| `reason`           | string  | Human-readable explanation           |
| `compliance_event` | object  | Structured event for the audit trail |

## Integration Points

- Reads KYC data via `card_platform_service` → `get_user_kyc_status()`
- Publishes compliance events consumed by the **LedgerAgent**
- References policies defined in `foundry/schemas/compliance.schema.json`

## Configuration

See `foundry/schemas/compliance.schema.json` for the full configuration schema,
including threshold values, tier requirements, and reporting intervals.
